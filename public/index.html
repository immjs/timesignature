Right, but how does it really work?


Signature

We get the hash of the file

We encrypt the timestamp with the file hash so data exposed by more extended usages of the protocol dont get leaked if a tsig file gets lost

We return a time signature

We get the timestamp (other data may be added by extended uses of the protocol)

We get our private key


Verification

We get the timestamp (other data may be added by extended uses of the protocol)

We get the file hash

We get the authority of the signature

We get the authority's public key from DNS or HTTP(S)

We unlock the second layer of encryption

We unlock the first layer of encryption and get our original data
